# 语音功能

## 概述

这个项目支持一种接近真实通话体验的语音对话功能，基于亚马逊的 **Amazon Nova Sonic** 模型（`amazon.nova-2-sonic-v1:0`）。用户对着手机说话，AI 能够同时接收音频、返回语音回复和文字转写，而且回复是实时语音输出，不是等说完再统一播放。

这套语音对话能力目前只有 iOS 和 Mac 版本有，Android 端尚未实现。

---

## 功能体验

在聊天界面，当用户选择了 Nova Sonic 作为文本模型、且处于文字模式下，输入框为空时，右侧会显示一个麦克风图标而不是发送按钮。点击这个按钮就开始语音对话。

使用流程是这样的：用户按住说话，音频会实时流式传给 Nova Sonic，AI 收到后一边生成回复一边直接用语音说出来，同时文字转写也会实时显示在屏幕上。如果 AI 在说话的过程中用户又开始说话，AI 会停下来听用户讲，这就是"打断"（barge-in）能力。整个对话延迟比较低，更接近真实打电话的体验，而不是"录一句、发过去、等回复、再播录音"这种异步模式。

在界面上，还有一个波形动画组件（AudioWaveformComponent）用来显示当前是输入状态还是输出状态，让用户知道现在是谁在说话。

---

## 语音设置

在 Settings 页面选择 Text Model 为 Nova Sonic 之后，会出现一个 Voice ID 下拉框，提供 12 种语音供选择：Matthew 和 Tiffany（美式英语）、Amy（英式英语）、Ambre 和 Florian（法语）、Beatrice 和 Lorenzo（意大利语）、Lennart 和 Greta（德语）、Lupe 和 Carlos（西班牙语）。

系统还给语音模式内置了几种预设提示词，包括 LearnWords（英语词汇教练，带发音评分）、LearnSentences（英语发音教练）和 Story（讲故事专家），这些 prompt 在 Constants.ts 中定义。

## 代码架构

整个语音功能的代码分为三个层次。

### 前端 UI 层

这是 React Native 的部分，iOS 和 Android 都有，包括麦克风按钮（CustomSendComponent）、音频波形动画（AudioWaveformComponent）、语音模式下的聊天界面集成（ChatScreen）以及设置页面中的 Voice ID 下拉框（SettingsScreen）。UI 层不负责音频处理，只是调用原生模块。

```typescript
// CustomSendComponent.tsx - 麦克风按钮的显示条件
const showMicButton =
  isNovaSonic &&                           // 选择了 Nova Sonic 模型
  mode === ChatMode.Text &&                // 当前是文字模式
  !text.trim() &&                          // 输入框为空
  selectedFiles.length === 0 &&            // 没有附件
  chatStatus !== ChatStatus.Running;       // 语音对话未在进行中
```

### 原生音频层

这里是真正干活的部分，负责麦克风音频捕获、流式传输给亚马逊、从亚马逊接收音频并实时播放。这套逻辑完全用 iOS 原生代码实现，调用了 AVFoundation 和 AVAudioEngine 框架，目前 Android 没有对应的实现。

涉及的文件有：
- `VoiceChatModule.swift` — React Native 桥接层，把原生能力暴露给 JS
- `AudioManager.swift` — 核心音频引擎，负责麦克风捕获和扬声器播放，音频格式为 16kHz PCM 输入、24kHz 输出
- `NovaSonicService.swift` — AWS Bedrock Nova Sonic API 客户端，处理双向流式通信
- `ConversationManager.swift` — 会话状态协调器，管理整个语音对话的生命周期

### 云端层

调用的是亚马逊 AWS Bedrock 的 Nova Sonic 接口，这一层跟平台无关，iOS 和 Android 都能调，只是 Android 还没写调用的那套原生代码。

---

## 为什么不支持 Android

整个语音功能依赖的是 iOS 原生平台的音频能力，需要用 iOS 的 `AVFoundation` 和 `AVAudioEngine` 实时处理音频流。这套代码完全没有跨平台实现，是专门为 iOS 写的。Android 端没有任何对应的原生模块，所以语音功能在 Android 上是完全缺失的状态。

具体来说，SettingsScreen 中有一段代码在 Android 上把 Nova Sonic 模型过滤掉了：

```typescript
if (Platform.OS === 'android') {
  bedrockResponse.textModel = bedrockResponse.textModel.filter(
    model => model.modelName !== 'Nova Sonic'
  );
}
```

这意味着 Android 用户在模型列表里根本看不到 Nova Sonic，不是说能看到但点不了。

## 相关文件一览

### JS/TS 端

- `src/chat/service/VoiceChatService.ts` — JS 桥接层，封装对原生模块的调用
- `src/chat/component/AudioWaveformComponent.tsx` — 音频波形可视化
- `src/chat/component/CustomSendComponent.tsx` — 麦克风按钮入口
- `src/chat/ChatScreen.tsx` — 语音模式集成
- `src/settings/SettingsScreen.tsx` — Voice ID 下拉框及 Android 过滤逻辑
- `src/storage/Constants.ts` — 语音 ID 列表和预设提示词

### iOS 原生端

- `ios/Modules/VoiceChat/VoiceChatModule.swift` — React Native 桥接
- `ios/Modules/VoiceChat/VoiceChatModule.m` — Obj-C 桥接头
- `ios/Services/AudioManager.swift` — 音频捕获与播放
- `ios/Services/NovaSonicService.swift` — AWS API 客户端
- `ios/Services/ConversationManager.swift` — 会话状态管理

### 权限配置

- `ios/SwiftChat/SwiftChat.entitlements` — `com.apple.security.device.audio-input`
- `Info.plist` — `NSMicrophoneUsageDescription`（用途描述为"支持录视频、语音聊天、内容总结"）
