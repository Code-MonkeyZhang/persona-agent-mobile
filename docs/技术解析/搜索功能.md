# SwiftChat 搜索功能技术解析

## 概述

SwiftChat 的搜索功能允许 AI 在回答问题时联网查询最新信息，支持多种搜索 Provider，包括 Tavily、Google、Bing 和 Baidu。当用户发送的消息需要最新信息时，系统会自动触发搜索流程。

---

## 搜索完整流程（以 Google/Bing/Baidu 为例）

### 第一步：用户发送消息，AI 判断是否需要搜索

当用户在对话框里发送一条消息时，系统会先用 AI 分析一下："这条消息是否需要联网查询最新信息？"。比如用户问"今天北京的天气怎么样"，AI 知道这需要最新数据，就会触发搜索流程。如果是"量子纠缠是什么"这种纯知识类问题，AI 可能就直接回答了，不会走搜索。

这个判断是通过 `IntentAnalysisService.ts` 实现的，它会分析用户消息和对话历史，提取搜索关键词和直接 URLs。

**IntentAnalysisService.ts（第 13-79 行）- AI 意图分析提示词：**

```typescript
const INTENT_ANALYSIS_PROMPT = `You are an AI question rephraser. Your role is to rephrase follow-up queries from a conversation into a standalone search query that can be used to retrieve information through web search.

## Guidelines:
1. If the question is a simple writing task, greeting, or general conversation, set "need_search" to false
2. If the user asks about specific URLs, include them in the "links" array
3. Extract ONE most comprehensive and appropriate search keyword into the "question" field
...

## Output Format:
{
  "need_search": boolean,
  "question": string,
  "links": string[]
}
```

系统会把这个提示词和对话历史一起发送给 AI，AI 返回一个 JSON，格式如 `{"need_search": true, "question": "北京天气", "links": []}`。

### 第二步：提取搜索关键词

如果 AI 判断需要搜索，它会从用户消息里提取出搜索关键词。比如问"特斯拉今天股价多少"，AI 会提取"特斯拉 股价"这样的关键词，然后用这个关键词去构造搜索引擎的 URL。

### 第三步：打开一个隐藏的 WebView

这一步就是"隐藏 WebView 模拟搜索引擎"。`WebViewSearchService.ts` 是核心，它负责创建隐藏的 WebView 并注入 JavaScript 来提取搜索结果。

**WebViewSearchService.ts（第 249-293 行）- WebView 搜索核心逻辑：**

```typescript
async search(
  query: string,
  engine: SearchEngine = 'google',
  maxResults: number = 5,
  abortController?: AbortController
): Promise<SearchResultItem[]> {
  this.currentEngine = engine;
  this.currentQuery = query;

  return new Promise((resolve, reject) => {
    // ...

    this.messageCallback = (message: WebViewMessage) => {
      // 收到 WebView 发来的消息
      if (message.type === 'search_results') {
        const provider = this.getProvider(engine);
        const results = provider.parseResults(message);
        resolve(results.slice(0, maxResults));
      }
    };

    // 获取对应的 Provider（Google/Bing/Baidu）
    const provider = this.getProvider(engine);
    const searchUrl = provider.getSearchUrl(query);  // 构造搜索 URL

    // WebView 页面加载完成后，注入提取脚本
    this.addEventListener('webview:loadEndTriggered', () => {
      const script = provider.getExtractionScript(query);
      this.sendEvent('webview:injectScript', { script });  // 注入 JS
    });

    // 让 WebView 加载搜索 URL
    this.sendEvent('webview:loadUrl', { url: searchUrl });
  });
}
```

每个 Provider 会提供自己的搜索 URL 构造方式和结果提取脚本：

**GoogleProvider.ts（第 20-23 行）- 构造搜索 URL：**

```typescript
getSearchUrl(query: string): string {
  const encodedQuery = encodeURIComponent(query);
  return `https://www.google.com/search?q=${encodedQuery}`;
}
```

**GoogleProvider.ts（第 25-165 行）- 注入到 WebView 的 JavaScript 提取脚本：**

```javascript
(function() {
  try {
    // 检查是否被重定向到 CAPTCHA 页面
    if (window.location.href.includes('/sorry/')) {
      window.ReactNativeWebView.postMessage(JSON.stringify({
        type: 'captcha_required',
        message: 'Google CAPTCHA verification required'
      }));
      return;
    }

    const results = [];
    // 尝试多种选择器来匹配 Google 搜索结果的 DOM 结构
    const selectors = [
      '#search .MjjYud',
      '#search .g',
      '#rso .g',
      '.hlcw0c',
      '[data-sokoban-container]',
      // ...
    ];

    for (const selector of selectors) {
      items = document.querySelectorAll(selector);
      if (items && items.length > 0) break;
    }

    // 从每个结果项中提取标题和链接
    items.forEach((item) => {
      const titleElement = item.querySelector('h3');
      const linkElement = item.querySelector('a');
      if (titleElement && linkElement && linkElement.href) {
        results.push({
          title: titleElement.textContent.trim(),
          url: linkElement.href
        });
      }
    });

    // 把提取结果发回给 React Native
    window.ReactNativeWebView.postMessage(JSON.stringify({
      type: 'search_results',
      results: results
    }));
  } catch (error) {
    window.ReactNativeWebView.postMessage(JSON.stringify({
      type: 'search_error',
      error: error.message
    }));
  }
})();
```

**本质上就是**：用 WebView 假装成一个浏览器访问了搜索引擎，然后通过注入的 JavaScript 从页面 HTML 里"爬"出搜索结果。

每个搜索 Provider 有不同的提取逻辑：

- **GoogleProvider**：从 `#search .MjjYud`、`#search .g`、`h3` 等元素提取，处理 CAPTCHA 检测
- **BingProvider**：从 `#b_results h2` 提取，处理编码 URL 解码
- **BaiduProvider**：从 `#content_left .result h3 a` 提取

### 第四步：抓取网页实际内容

搜到一堆链接后，系统会挑前几个最相关的，然后去抓取这些网页的实际内容。这步用的是 Mozilla 的 Readability 库，它能把网页的 HTML 转成干净的纯文本。转换后的内容会通过 TurndownService 进一步转成 Markdown 格式，方便给 AI 当参考资料。

**ContentFetchService.ts（第 141-170 行）- 使用 Readability 解析网页：**

```typescript
// 使用 Readability 解析网页内容
const { document } = parseHTML(html, { url: finalUrl });
const reader = new Readability(document);
const article = reader.parse();

// 使用 TurndownService 把 HTML 转成 Markdown
const turndownService = new TurndownService();
const contentParsed = parseHTML(htmlContent);
const contentDoc = contentParsed.document;
const markdownContent = turndownService.turndown(contentDoc);
```

系统会并发抓取多个页面，但会在前 3-5 个有效结果后提前终止，避免浪费时间：

```typescript
// 最多等前 3 个"Top 3"结果返回就终止
if (top3Count === 3 && totalCompleted >= 3) {
  globalAbortController.abort();
  break;
}
```

### 第五步：把搜索结果塞给 AI

系统会把搜到的内容整理成一个增强版的提示词，包含：
- 当前时间戳
- AI 回复的引用规则
- 用户原始问题
- 参考资料（标题 + URL + 内容摘要）

然后再把整个提示词发给 AI 模型，让它基于这些真实搜索结果来回答用户的问题。

**WebSearchOrchestrator.ts（第 121-144 行）- 构建增强版提示词和引用：**

```typescript
onPhaseChange?.(WebSearchPhase.BUILDING);

// 构建增强版提示词
const enhancedPrompt = promptBuilderService.buildPromptWithReferences(
  userMessage,
  contents
);

// 构造系统提示词
const webSearchSystemPrompt: SystemPrompt = {
  id: -999,
  name: 'Web Search References',
  prompt: enhancedPrompt,
  includeHistory: true,
};

// 构造引用列表
const citations: Citation[] = contents.map((content, index) => ({
  number: index + 1,
  title: content.title,
  url: content.url,
  excerpt: content.excerpt,
}));

return {
  systemPrompt: webSearchSystemPrompt,
  citations: citations,
};
```

## Tavily Provider 的区别

Tavily 是一个专门的搜索 API，它干的事情和上面第三步到第四步一模一样，但是是通过 API 调用直接返回结果，不需要用 WebView"爬"页面。所以 Tavily 更快更稳定，但需要用户有一个 Tavily 的 API Key。

| Provider | 搜索方式 | API Key | CAPTCHA | 内容获取 |
|----------|---------|---------|---------|---------|
| Tavily | 直接 API 调用 | 需要 | 无 | API 返回包含内容 |
| Google | WebView 爬取 | 不需要 | 容易遇到（120s 超时） | 额外抓取阶段 |
| Bing | WebView 爬取 | 不需要 | 无 | 额外抓取阶段 |
| Baidu | WebView 爬取 | 不需要 | 无 | 额外抓取阶段 |

## 整体 orchestration（编排器）

`WebSearchOrchestrator.ts` 是整个搜索流程的编排器，它串联了所有阶段：

**WebSearchOrchestrator.ts（第 47-155 行）- execute 方法完整流程：**

```typescript
async execute(
  userMessage: string,
  bedrockMessages: BedrockMessage[],
  onPhaseChange?: (phase: string) => void,
  searchEngine?: SearchEngine,
  abortController?: AbortController
): Promise<WebSearchResult | null> {
  // 1. 获取当前选择的 Provider
  const providerOption = searchEngine || (getSearchProvider() as SearchEngineOption);
  if (providerOption === 'disabled') return null;

  // 2. Phase 1: AI 判断是否需要搜索
  onPhaseChange?.(WebSearchPhase.ANALYZING);
  const intentResult = await intentAnalysisService.analyze(userMessage, bedrockMessages);
  if (!intentResult.needsSearch || intentResult.keywords.length === 0) return null;

  // 3. Phase 2: 执行搜索
  onPhaseChange?.(WebSearchPhase.SEARCHING);
  const keyword = intentResult.keywords[0];

  let contents;
  if (engine === 'tavily') {
    // Tavily: 直接 API 调用
    contents = await tavilyProvider.search(keyword, 5, abortController);
  } else {
    // Google/Bing/Baidu: WebView 爬取
    const searchResults = await webViewSearchService.search(keyword, engine, 8, abortController);
    if (searchResults.length === 0) return null;

    // 4. Phase 3: 抓取网页内容
    onPhaseChange?.(WebSearchPhase.FETCHING);
    contents = await contentFetchService.fetchContents(searchResults, 8000, 10000, abortController);
  }

  if (contents.length === 0) return null;

  // 5. Phase 4: 构建增强提示词
  onPhaseChange?.(WebSearchPhase.BUILDING);
  const enhancedPrompt = promptBuilderService.buildPromptWithReferences(userMessage, contents);
  // ... 返回 systemPrompt 和 citations
}
```

## 引用系统

AI 返回的回复会带上引用标记，格式为 `[1]`、`[2]`、`[3]` 等。每个数字对应一个具体的参考资料。用户点击引用标记可以查看详细信息，包括网页标题、URL 和内容摘要。

引用数据通过 `Citation` 接口定义：

```typescript
interface Citation {
  number: number;    // 引用编号 [1], [2], [3]
  title: string;     // 链接标题
  url: string;       // 链接 URL
  excerpt?: string;  // 内容摘要
}
```
