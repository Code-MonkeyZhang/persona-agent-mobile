# 屏幕自适应

## 概述

这个 React Native 项目没有使用任何第三方响应式布局库，完全依靠以下原生能力实现跨设备适配：

- React Native 内置的 `Dimensions` API 检测屏幕尺寸
- `Platform` 模块区分 iOS/Android
- 原生模块区分 Mac Catalyst 桌面端
- 多个硬编码阈值在不同设备形态间做分支

---

## 平台检测

### 文件

`src/utils/PlatformUtils.ts`

### 实现

```typescript
export const isMacCatalyst = Platform.OS === 'ios' && PlatformModule?.isMacCatalyst === true;
export const isAndroid = Platform.OS === 'android';
```

在 `App.tsx` 中全局导出：

```typescript
export const isMac = isMacCatalyst;
```

`isMac` 是整个应用最大的自适应开关，Mac 端会使用固定宽度布局（抽屉常驻 300px），而手机端使用弹性布局。

## 屏幕尺寸获取

### 文件

`src/chat/ChatScreen.tsx`

### 实现

直接调用 `Dimensions.get('window')` 并通过 `addEventListener` 监听屏幕旋转等尺寸变化事件：

```typescript
const [screenDimensions, setScreenDimensions] = useState(Dimensions.get('window'));

useEffect(() => {
  const updateDimensions = () => {
    setScreenDimensions(Dimensions.get('window'));
  };

  const subscription = Dimensions.addEventListener('change', updateDimensions);
  return () => subscription?.remove();
}, []);
```

没有封装成自定义 hook，每个需要的页面各自管理。

## 横竖屏检测

通过 `screenHeight < screenWidth` 判断是否横屏。

### 滚动到底部按钮

```typescript
const scrollStyle = StyleSheet.create({
  scrollToBottomContainerStyle: {
    width: 30,
    height: 30,
    left:
      Platform.OS === 'ios' &&
      screenHeight < screenWidth &&      // 横屏
      screenHeight < 500                  // 小屏手机横屏（iPhone）
        ? screenWidth / 2 - 75            // 居中偏移
        : chatScreenWidth / 2 - 15,
    bottom: screenHeight > screenWidth ? '1.5%' : '2%',  // 竖屏贴底，横屏往上
  },
});
```

横竖屏切换时，按钮的水平位置和底部距离都会重新计算。

## 大屏手机 vs 小屏手机

### 关键阈值

| 阈值 | 用途 |
|------|------|
| `width > 434` | 图片画廊 5 列 vs 3 列 |
| `screenWidth < 500` | iPhone 竖屏键盘偏移调整 |
| `screenHeight < 500` | iPhone 横屏滚动按钮定位 |
| `minWidth > 434` | Drawer 固定 300px vs 屏幕宽度的 83% |

### Drawer 宽度（App.tsx）

```typescript
const { width: screenWidth, height: screenHeight } = Dimensions.get('window');
const minWidth = screenWidth > screenHeight ? screenHeight : screenWidth;
const width = minWidth > 434 ? 300 : minWidth * 0.83;
```

取宽高中的较小值作为参考轴。如果这个最小值大于 434px（平板级别），Drawer 用固定 300px；否则用屏幕较小边的 83%。

### 图片画廊列数（ImageGalleryScreen.tsx）

```typescript
const getNumColumns = (width: number) => (width > 434 ? 5 : 3);
```

### 键盘偏移（ChatScreen.tsx）

```typescript
bottomOffset={
  Platform.OS === 'android'
    ? 0
    : screenHeight > screenWidth && screenWidth < 500
    ? 24  // iPhone 竖屏
    : 12
}
```

### 最大输入框高度

```typescript
maxComposerHeight={isMac ? 360 : 200}
```

## 键盘处理

### 键盘弹出时自动滚动到底部

```typescript
useEffect(() => {
  const handleKeyboardShow = () => {
    if (textInputViewRef.current?.isFocused()) {
      scrollToBottom();
    }
  };

  const keyboardDidShowListener = Platform.select({
    ios: Keyboard.addListener('keyboardWillShow', handleKeyboardShow),
    android: Keyboard.addListener('keyboardDidShow', handleKeyboardShow),
  });

  return () => {
    keyboardDidShowListener && keyboardDidShowListener.remove();
  };
}, []);
```

iOS 用 `keyboardWillShow`，Android 用 `keyboardDidShow`，平台分支处理。

### 页面切换时收起键盘

```typescript
<NavigationContainer
  onStateChange={_ => {
    Keyboard.dismiss();
  }}>
```

### PromptScreen 自动调整键盘内边距

```typescript
<ScrollView
  style={styles.container}
  automaticallyAdjustKeyboardInsets={true}>
```

## SafeArea 处理

### 使用 react-native-safe-area-context

```typescript
import { useSafeAreaInsets } from 'react-native-safe-area-context';

const insets = useSafeAreaInsets();
const statusBarHeight = useRef(insets.top);
```

### CustomChatFooter 中的偏移

```typescript
iconPositionRef.current = {
  x: pageX,
  y: pageY + 10 + (isAndroid ? statusBarHeight.current : 0),
};
```

Android 需要额外加上 status bar 高度，iOS 则不需要。

### Mac 端条件化 SafeArea

```typescript
<SafeAreaView style={[isMac ? styles.macContainer : styles.safeArea]}>
```

## 样式系统

项目使用 React Native 内置的 `StyleSheet.create()`，这是 React Native 标准 CSS-in-JS 方案，没有引入 styled-components 等第三方库。

### 布局方式总结

| 方式 | 示例 |
|------|------|
| Flexbox | `flex: 1`、`flexDirection: 'row'` |
| 固定值 | 图标 24x24、按钮 44x44 |
| 百分比 | WebView 内容 `width: '100%'` |
| 屏幕尺寸计算 | `screenWidth - 300`（Mac 抽屉占位） |
| 平台分支 | `Platform.select({ ios: ..., android: ... })` |

---

## 主题系统

配合响应式布局，项目有一套完整的主题系统：

- `ThemeProvider` 包裹应用
- `useTheme()` hook 提供 `colors` 和 `isDark`
- 通过 `useColorScheme()` 跟随系统深色/浅色模式
- Mac 端有专属颜色变量（如 `selectedBackgroundMac`）
