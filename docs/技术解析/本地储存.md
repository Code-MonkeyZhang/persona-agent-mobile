# 本地存储

## 存储方案

这个 App 使用 **MMKV**（全称 react-native-mmkv）作为本地存储方案。MMKV 是一个高性能的键值存储库，来自微信团队，相比 React Native 中常用的 AsyncStorage，它的读写速度更快、性能更好。

项目里实际存在两个 MMKV 存储实例：

- `storage` - 普通的存储实例，用来放聊天记录、模型配置、App 设置这些不敏感的数据
- `encryptStorage` - 加密的存储实例，用 UUID 作为密钥，专门用来存放 API Key、Token 这类敏感信息，防止被轻易读取

相关的代码主要在三个地方：

```typescript
// src/storage/StorageUtils.ts
// 主存储工具文件，约 980 行，定义了所有 get/set/delete 相关的函数

// src/storage/Constants.ts
// 存放默认值、模型列表、区域列表、系统提示词模板等常量

// src/chat/util/FaviconUtils.ts
// Favicon 服务的缓存逻辑，缓存有效期是 7 天
```

## 普通存储里放了什么

### 聊天记录

聊天相关的数据是本地存储里最重要的一部分。每个聊天会话都有自己的唯一 ID，会话列表和具体消息是分开存储的。

- `bedrock/messageList` - 存储所有聊天会话的元信息，包括每个会话的 id、标题、模式（文本/语音/图片）和创建时间
- `bedrock/sessionId/{id}` - 根据会话 ID 存储具体的消息内容，每条消息包含发送者、内容、时间戳等信息
- `bedrock/currentSessionId` - 记录当前正在聊天的那个会话 ID

举个例子，一个 messageList 大概长这样：

```typescript
[
  {
    id: 1,
    title: "关于 React Native 的讨论",
    mode: "text",
    timestamp: 1699000000000
  },
  {
    id: 2,
    title: "图片生成任务",
    mode: "image",
    timestamp: 1699100000000
  }
]
```

### API 配置

这里存放各种模型服务的连接信息，包括地址和区域配置：

- `bedrock/apiUrlKey` - 主 API 的地址
- `bedrock/ollamaApiUrlKey` - Ollama服务的 API 地址
- `bedrock/openAICompatApiURLKey` - OpenAI 兼容接口的地址
- `bedrock/openAICompatModelsKey` - OpenAI 兼容接口可用的模型列表，用逗号分隔
- `bedrock/regionKey` - AWS 区域，默认是 `us-west-2`
- `bedrock/bedrockConfigModeKey` - 当前配置模式，取值是 `bedrock` 或 `swiftchat`

### 模型设置

用户选择的模型和相关配置都在这里：

- `bedrock/textModelKey` - 当前选中的文本生成模型
- `bedrock/imageModelKey` - 当前选中的图片生成模型
- `bedrock/allModelKey` - 所有可选的文本和图片模型，这个是完整的模型池
- `bedrock/imageSizeKey` - 图片生成的尺寸，比如 `1024 x 1024`
- `bedrock/voiceIdKey` - 语音聊天时选中的语音 ID
- `bedrock/modelOrderKey` - 模型的排序，使用越频繁的模型排得越靠前，方便用户快速选择
- `bedrock/modelUsageKey` - 记录每个模型的使用量，比如消耗了多少 Token、生成了几张图片

### System Prompts

System Prompt 是发给 AI 的指令，决定 AI 的行为和角色设定。用户可以创建多个自定义提示词：

- `bedrock/systemPromptsKey` - 存储用户创建的所有自定义系统提示词
- `bedrock/currentSystemPromptKey` - 当前激活的文本对话提示词
- `bedrock/currentVoiceSystemPromptKey` - 当前激活的语音对话提示词
- `bedrock/currentImageSystemPromptKey` - 当前激活的图片生成提示词
- `bedrock/currentPromptIdKey` - 记录当前选中的是哪个提示词

### App 设置

一些偏好和开关设置：

- `bedrock/hapticEnabledKey` - 触感反馈开关，默认开启
- `bedrock/openAIProxyEnabledKey` - 是否启用 OpenAI 代理
- `bedrock/thinkingEnabledKey` - 思考/推理模式开关，默认开启
- `bedrock/reasoningExpandedKey` - AI 推理过程是否展开显示，默认展开
- `bedrock/searchProviderKey` - 网页搜索的提供商，可以是 `disabled`（关闭）或 `tavily`
- `bedrock/lastVirtualTryOnImgFileTag` - 虚拟试穿功能用的，记录上一次使用的底图信息

### App Builder

App Builder 是 App 内置的一个功能，让用户可以自己构建 Web App。构建好的 App 会保存起来：

- `bedrock/savedAppsKey` - 保存的 App 元数据列表
- `app_code_{appId}` - 每个 App 的 HTML 代码，按 appId 索引

### Favicon 缓存

- `favicon_services_map` - 域名到 Favicon 服务的映射关系，缓存有效期 7 天，用来快速获取网站图标

## 加密存储里放了什么

加密存储专门用来放 API Key 和 Token 这类敏感数据，防止被恶意读取或泄露：

- `bedrock/apiKeyTag` - 主 API Key
- `bedrock/ollamaApiKeyTag` - Ollama 的 API Key
- `bedrock/deepSeekApiKeyTag` - DeepSeek 的 API Key
- `bedrock/openAIApiKeyTag` - OpenAI 的 API Key
- `bedrock/openAICompatApiKeyTag` - OpenAI 兼容接口的 API Key
- `bedrock/bedrockApiKeyTag` - AWS Bedrock 的凭证
- `bedrock/tavilyApiKeyTag` - Tavily 搜索服务的 API Key
- `bedrock/tokenInfo` - AWS STS 的临时凭证，包含 accessKeyId、secretAccessKey、sessionToken 和过期时间
- `bedrock/openAICompatConfigsKey` - OpenAI 兼容接口的额外配置

这些数据加密存储后，即使手机被 root 或越狱，攻击者也很难直接拿到明文的 Key。

## 常用操作

StorageUtils.ts 封装了很多便捷函数，常用的有：

**聊天记录**
```typescript
saveMessages(sessionId, messages)
getMessagesBySessionId(sessionId)
deleteMessagesBySessionId(sessionId)
saveMessageList(chatList)
getMessageList()
```

**模型相关**
```typescript
saveTextModel(model)
getTextModel()
saveImageModel(model)
getImageModel()
saveModelOrder(order)
getModelOrder()
updateTotalUsage(usage)
```

**设置相关**
```typescript
saveHapticEnabled(enabled)
getHapticEnabled()
saveThinkingEnabled(enabled)
getThinkingEnabled()
```

**清理**
```typescript
clearAllChatHistory()  // 清除所有聊天记录和保存的 App
```
