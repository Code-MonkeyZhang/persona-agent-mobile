# SwiftChat 产品需求文档

## 产品背景

SwiftChat 是一款跨平台的 AI 助手应用，旨在为用户提供一个便捷的入口来使用各种主流 AI 大模型。随着大模型技术的快速发展，用户面临着模型选择繁多、切换困难、使用门槛高等问题。SwiftChat 通过整合 Amazon Bedrock、Ollama、DeepSeek、OpenAI 等多种 AI 能力，让用户可以在一个应用中完成大多数 AI 相关的任务，而不需要在多个应用之间来回切换。

除了提供基础的对话能力之外，SwiftChat 还有一个独特的功能——即时 Web 应用创建。用户只需要用自然语言描述自己想要的应用，AI 就能自动生成一个可以运行的 Web 应用，并支持在应用内预览、编辑和分享。这个功能极大地降低了非技术人员创建 Web 应用的门槛，让每个人都能快速将想法变成可用的产品。

## 目标用户

SwiftChat 的主要用户群体可以分为几类。第一类是普通的日常用户，他们希望有一个简单的方式来获取 AI 的帮助，比如查询信息、写作辅助、翻译等。第二类是开发者和技术爱好者，他们需要频繁切换不同的 AI 模型来对比效果，或者使用本地部署的 Ollama 来节省成本。第三类是内容创作者，他们需要 AI 帮助生成图像、撰写文案，或者快速原型化一个 Web 应用来展示自己的创意。

## 产品功能概述

### 对话功能

对话是 SwiftChat 最核心的功能。用户进入应用后首先看到的是对话列表页面，这里展示了所有历史对话记录，每条记录都显示了对话的标题和最后活跃时间。用户可以通过侧边抽屉来管理对话列表，包括创建新对话、删除对话、对对话进行重命名等操作。

聊天界面是用户与 AI 交互的主要场所。用户可以在底部的输入框中输入文字内容，也可以上传图片或文件。输入框上方有一系列辅助按钮，包括语音输入、图片上传、文件上传等。发送消息后，AI 的回复会以流式方式逐渐显示在屏幕上，让用户能够第一时间看到 AI 的思考过程。所有的消息都支持富文本渲染，包括代码块、表格、LaTeX 数学公式、Mermaid 图表等常见格式。

在对话过程中，用户可以随时切换不同的 AI 模型。模型切换按钮位于聊天界面的顶部，用户可以从 Amazon Bedrock、Ollama、DeepSeek、OpenAI 等多种模型中选择自己需要的。选择不同的模型会影响后续所有对话的 AI 行为，但不会影响已经产生的对话历史。

### 即时 Web 应用创建

这是 SwiftChat 从 2.7.0 版本开始重点推的功能。它的核心理念是让用户只需要描述自己的想法，就能得到一个可以运行的 Web 应用。

整个流程从用户点击"创建应用"开始。用户会被引导到一个描述页面，在这里用自然语言描述自己想要创建的应用。比如，用户可以说"帮我创建一个待办事项列表应用，要有添加、删除、完成这些功能，配色要用蓝色系"。描述完成后，用户提交需求，AI 会根据描述生成一个完整的 Web 应用。

生成的应用会在应用画廊中展示。用户可以点击任何一个已创建的应用来查看详情。在详情页面中，应用会在一个 WebView 中直接运行，用户可以直接体验应用的功能。如果用户觉得某个地方需要调整，可以点击编辑按钮进入代码编辑模式，修改后再保存。如果用户想要分享给朋友，可以一键生成分享链接。

### 网络搜索

当用户的问题需要最新的信息时，SwiftChat 提供了网络搜索功能。用户可以在提问时开启搜索选项，AI 会先调用搜索工具获取最新的相关信息，然后结合搜索结果来回答用户的问题。

搜索功能支持多个搜索 Provider，包括 Tavily、Google、Baidu 和 Bing。用户可以在设置中选择默认使用的搜索 Provider。不同的 Provider 在不同地区和不同类型的问题上可能有不同的效果，用户可以根据自己的需求选择最合适的。

### AI 图像生成

SwiftChat 支持调用 Amazon Nova Canvas 模型来进行图像生成。用户可以使用文字描述来生成图片，也可以对已有图片进行虚拟试穿、背景去除等操作。

生成的图像会自动保存在图像画廊中。用户可以在画廊中浏览自己生成过的所有图片，也可以对图片进行删除或重新生成。如果用户想要将某张图片分享给朋友，可以直接导出到系统相册或生成分享链接。

### 语音对话

对于不喜欢打字的用户，SwiftChat 提供了语音对话功能。用户可以点击语音输入按钮，然后对着手机说话。语音会被实时转换为文字，并发送给 AI 处理。AI 的回复同样支持语音播报，用户可以选择让 AI 用语音来回答问题。

语音对话功能底层使用的是 Amazon Nova Sonic 模型，它能够实现低延迟的实时语音交互，接近人类对话的自然体验。

### 设置与个性化

在设置页面中，用户可以对自己的使用偏好进行各种配置。

首先是模型配置。用户需要在这里填入自己的 API Key 才能使用相应的 AI 服务。应用会加密存储这些敏感信息，不会将其上传到任何第三方服务器。除了 API Key 之外，用户还可以设置默认使用的模型、对话的最大长度等参数。

其次是系统提示词管理。用户可以为 AI 设置一个系统级别的指令，用来定义 AI 的角色和行为。比如，用户可以设定 AI 是一个专业的程序员、一个温柔的客服、或者一个严谨的学术助手。应用内置了一些常用的预设，用户也可以创建自己的自定义预设。

主题方面，SwiftChat 支持深色和浅色两种模式，用户可以根据自己的喜好或者当前的环境来切换。深色模式在夜间使用时更加护眼，也能节省 OLED 屏幕的电量。

最后，用量统计功能会记录用户使用各个模型的 Token 消耗情况，帮助用户了解自己的使用成本。用户可以在 Token 使用页面查看详细的统计信息。

## 界面交互说明

### 导航结构

SwiftChat 使用抽屉式导航作为主要导航方式。从屏幕左侧边缘向右滑动可以打开侧边抽屉，里面包含了应用的主要入口，包括对话列表、新建对话、创建应用、应用画廊、图像画廊、设置等。点击相应的入口会跳转到对应的页面，同时抽屉会自动关闭。

在具体的子页面中，导航栏通常会提供返回按钮、页面标题，以及一些页面特有的操作按钮。比如在对话页面，导航栏会显示当前对话的标题和模型切换按钮。

### 消息输入

消息输入框位于屏幕底部，始终可见。输入框支持多行文本输入，当用户输入的内容超过一行时，输入框会自动扩展高度，最多扩展到屏幕高度的三分之一左右。超过这个限制后，输入框会变成可滚动状态。

输入框旁边有一个发送按钮，当输入框为空时这个按钮是禁用状态。当用户输入内容后，按钮会变为可用状态，点击后会发送消息并清空输入框。在发送按钮上方还有一个扩展菜单按钮，点击后会展开更多的输入选项，包括图片上传、文件上传、语音输入等。

### 流式响应展示

当 AI 正在生成回复时，界面上会显示一个加载动画，表示 AI 正在处理中。一旦 AI 开始输出内容，加载动画会消失，取而代之的是实际的内容。内容会逐字或逐段地出现在屏幕上，这种流式展示的方式让用户能够及时看到 AI 的思考进展。

如果 AI 的回复中包含了代码块，系统会自动检测并应用语法高亮。代码块右上角会有一个复制按钮，用户可以一键复制代码到剪贴板。对于包含 LaTeX 公式的消息，系统会调用数学公式渲染组件来正确显示这些公式。

### 应用预览

当用户创建了一个 Web 应用后，可以在应用详情页面直接预览。这个预览是在一个全屏的 WebView 中进行的，用户可以像使用普通浏览器一样与应用进行交互。如果应用需要网络请求，这些请求会通过一个代理通道来确保功能的正常运行。

预览界面下方提供了一排快捷按钮。用户可以点击编辑按钮来查看和修改应用的代码，可以点击刷新按钮来重新加载应用，也可以点击分享按钮来生成分享链接。

## 数据存储

SwiftChat 使用 MMKV 作为本地存储方案。相比传统的 AsyncStorage，MMKV 是一个高性能的键值存储库，官方宣称读写速度比 AsyncStorage 快 10 倍以上。这对于需要频繁读取对话历史和用户设置的应用来说非常重要，能够显著提升应用的响应速度。

对话历史会保存在本地，用户可以在任何时候查看之前的对话内容。应用的元数据，比如创建时间、访问频率等，也会记录在本地。对于需要跨设备同步的高级功能，目前版本还未支持，用户需要注意这一点。

## 性能与兼容性

SwiftChat 基于 React Native 0.74 版本开发，支持 iOS 13.0 及以上版本，以及 Android 7.0 及以上版本。应用默认使用 Hermes 引擎运行 JavaScript 代码，这能够提供更快的启动速度和更低的内存占用。

在动画和手势处理方面，应用使用了 react-native-reanimated 和 react-native-gesture-handler 来保证流畅的用户体验。所有需要长时间处理的任务都会尽量放在后台线程中执行，以避免阻塞主线程导致界面卡顿。

## 安全与隐私

用户的 API Key 和其他敏感信息会使用加密算法存储在本地。应用不会将用户的对话内容上传到任何第三方服务器，所有的 AI 请求都是直接从用户设备发送到相应的 AI 服务商。

对于用户生成的 Web 应用，分享链接中不会暴露任何用户的个人信息。应用的代码完全在云端运行，用户分享的只是应用的访问链接，而不是应用的源代码。

- 更丰富的应用编辑能力，包括支持更多的 Web 技术栈
- 对话导出功能，支持将对话导出为 PDF、Markdown 等格式
- 更多的 AI 模型支持，持续跟进最新的 AI 技术发展
- 团队协作功能，支持多人共同编辑和管理应用
